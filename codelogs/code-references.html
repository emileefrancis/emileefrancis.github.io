For this Code Log, I looked up CSS Flexbox, specifically the idea of using display: flex; to control layout. I chose this because Flexbox is one of the most useful CSS tools for organizing content on a page, especially for navigation bars, card layouts, and responsive design.

What did I look up?

I looked up CSS Flexbox and how display: flex; works.

What sites did I consult?

  • MDN: Flexbox
  • W3Schools: CSS Flexbox
  • CSS-Tricks: A Complete Guide to Flexbox
  • web.dev: Flexbox

What are the differences in the information presented?

MDN explains Flexbox in a more detailed and technical way. W3Schools is shorter and easier to scan quickly. CSS-Tricks is more visual and helps show how the properties relate to each other. web.dev teaches the topic in more of a lesson format and connects it to responsive design.

What features should a code reference page offer?

I think a good code reference page should have a clear definition, simple examples, syntax, visual demonstrations, and browser support information. It should also be easy to scan when someone is trying to solve a coding problem fast.

Do any of the resources stand out?

MDN stands out as the most complete and accurate. CSS-Tricks stands out because it is especially visual and easy to understand for layout. W3Schools is useful for beginners because it is simple, but it does not go as deep.
