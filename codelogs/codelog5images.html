Images

Images are used on websites to add visual information and help explain content. In web development, images can be displayed using HTML or CSS depending on how the image is being used.

Using the img tag vs background-image

The HTML <img> tag places an image directly into the page as part of the content. Because of this, it behaves like other elements in the layout and can include alt text for accessibility. 

<img src="cake.jpg" alt="Bavarian cream sponge cake">

The CSS property background-image places an image as the background of an HTML element instead of being actual page content. 

.box {
  background-image: url("cake.jpg");
}

The main difference is that the img tag is used when the image is important content on the page, while background images are usually decorative. Another difference is layout. An img element affects the page structure, while a background image stays inside the element it is attached to.

Other CSS Properties for Background Images

Besides background-image, other CSS properties affect how background images appear.

background-size controls how large the background image appears inside the element. 

background-size: cover;

background-repeat controls whether the image repeats across the background. 

background-repeat: no-repeat;

background-position controls where the image appears inside the element. 

background-position: center;

Dimensions vs Resolution

Dimensions describe the width and height of a bitmap image in pixels, while resolution describes how much visual detail is packed into that image space.

Image File Formats

GIF supports animation but has limited colors, so it works best for small animations.

PNG keeps high image quality and supports transparency, which makes it good for graphics and logos.

JPG/JPEG is commonly used for photographs because it compresses image size well.

WebP is a newer format that can keep good quality while making file sizes smaller for websites.

SVG is vector based, meaning it can scale to any size without losing quality, which makes it good for icons and logos.

These formats all store image data but differ in compression, transparency, animation support, and whether they are bitmap or vector images. Because of this, some formats are better for certain uses.
